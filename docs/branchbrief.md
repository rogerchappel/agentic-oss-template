# Branchbrief

Branchbrief is a lightweight pull request summary generated by GitHub Actions.

The baseline workflow creates a `branchbrief.md` artifact with:

- base branch
- head branch
- commit SHA
- recent commits
- changed files
- diff stat

It does not post comments, mutate pull requests, call external services, or
require secrets.

## How to use it

1. Open a pull request.
2. Wait for the `Branchbrief` workflow to finish.
3. Download the `branchbrief` artifact from the workflow run.
4. Use it as a quick review aid alongside the pull request diff.

## Customizing later

Projects may later add checks for ownership, risk labels, release notes, or
review-pack completeness. If the workflow starts writing comments or labels,
change permissions explicitly and document why write access is required.
